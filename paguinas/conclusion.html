SQL

SQL (por sus siglas en inglés Structured Query Language; en español lenguaje de consulta estructurada) es un lenguaje específico de dominio, diseñado para administrar, y recuperar información de sistemas de gestión de bases de datos relacionales.2​ Una de sus principales características es el manejo del álgebra y el cálculo relacional para efectuar consultas con el fin de recuperar, de forma sencilla, información de bases de datos, así como realizar cambios en ellas.

Mas información

Originalmente basado en el álgebra relacional y en el cálculo relacional, SQL consiste en un lenguaje de definición de datos, un lenguaje de manipulación de datos y un lenguaje de control de datos. El alcance de SQL incluye la inserción de datos, consultas, actualizaciones y borrado, la creación y modificación de esquemas y el control de acceso a los datos. También el SQL a veces se describe como un lenguaje declarativo, también incluye elementos procesales.

Mas información

SQL fue uno de los primeros lenguajes comerciales para el modelo relacional de Edgar Frank Codd como se describió en su artículo de investigación de 1970 El modelo relacional de datos para grandes bancos de datos compartidos. A pesar de no adherirse totalmente al modelo relacional descrito por Codd, pasó a ser el lenguaje de base de datos más usado.

CONCLUSION


El manejo de sentencias DDL (Data Definition Language) y DML (Data Manipulation Language) en SQL es fundamental para la administración y manipulación de bases de datos. A través de este ejercicio, hemos aprendido a realizar una serie de operaciones esenciales que abarcan desde la creación de una tabla hasta la actualización y consulta de registros. Utilizando phpMyAdmin, una herramienta gráfica para la gestión de bases de datos MySQL, estas operaciones se pueden llevar a cabo de manera intuitiva y eficiente.

Resumen de los Pasos Realizados:

1. Creación de la Tabla "libreta": Aprendimos a definir una estructura de tabla con campos específicos, utilizando la sentencia `CREATE TABLE`.
2. Visualización de Tablas Existentes: Utilizamos `SHOW TABLES` para verificar la existencia de nuestras tablas en la base de datos.
3. Descripción de la Estructura de una Tabla: Con `DESCRIBE`, examinamos la estructura y los tipos de datos de los campos en la tabla "libreta".
4. Inserción de Registros: Empleamos `INSERT INTO` para añadir datos a la tabla, lo cual es crucial para poblar la base de datos con información relevante.
5. Consulta de Registros: Utilizamos `SELECT *` para recuperar y visualizar todos los registros de la tabla, permitiendo así la verificación y análisis de los datos almacenados.
6. Actualización de Registros: Mediante `UPDATE`, demostramos cómo modificar los datos existentes en la tabla para reflejar cambios o correcciones necesarias.
7. Inserción de Múltiples Registros: Insertamos varios registros adicionales, mostrando cómo se puede expandir rápidamente la base de datos.
8. Conteo de Registros: Con `SELECT COUNT(*)`, contamos el número total de registros en la tabla, una operación común para obtener estadísticas básicas de la base de datos.

Importancia de DDL y DML:

- DDL (Data Definition Language): Permite definir y modificar la estructura de la base de datos. Las sentencias como `CREATE`, `ALTER`, y `DROP` son fundamentales para establecer y mantener la estructura de las tablas.
- DML (Data Manipulation Language): Facilita la manipulación de los datos almacenados. Las sentencias `INSERT`, `UPDATE`, `DELETE`, y `SELECT` son esenciales para gestionar la información que reside en la base de datos.

El uso efectivo de estas sentencias en conjunto permite una gestión robusta y flexible de las bases de datos, asegurando que los datos sean precisos, actualizados y organizados de manera lógica. PhpMyAdmin, con su interfaz gráfica, simplifica estas tareas y proporciona una plataforma accesible para usuarios de todos los niveles, desde principiantes hasta administradores de bases de datos experimentados.

En resumen, dominar las sentencias DDL y DML es crucial para cualquier persona que trabaje con bases de datos, y herramientas como phpMyAdmin hacen que estas operaciones sean más accesibles y manejables, permitiendo una administración de datos más eficiente y efectiva.
